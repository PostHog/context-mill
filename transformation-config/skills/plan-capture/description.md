# Plan what to capture

Read the app and decide which custom events are worth capturing. You are
planning, not editing.

Look for the actions that matter to a product team: signups, key feature use,
content created, purchases, sharing, settings changed. Skip low-value noise and
anything autocapture already covers (generic clicks, pageviews).

For each event choose a clear, consistent name — `lower_snake_case`, named for
the action — and note the file and the user action that should trigger it. A few
high-signal events beat a long list. Hand off the plan so the capture step can
instrument it.
